Findings

Critical line-height:44px on every <a> bloats vertical rhythm render.py:1340

The mobile touch-target rule sets a { min-height:44px; min-width:44px; display:inline-block; line-height:44px; } for all anchors. The override on the next line catches links inside <p>/<div>/<td>, but any bare <a> — section anchors, "see more" links in a heading, footer links — gets a 44px line-height that visually pushes surrounding text apart.

FixDrop line-height:44px, keep min-height:44px. The 44px height alone satisfies WCAG touch targets without breaking rhythm.
Critical min-font-size is not a real CSS property render.py:1335

The mobile block declares min-font-size:14px. This is not a valid CSS property — browsers silently drop it. The adjacent font-size:14px !important on the same selector is doing the actual work, but the intent was a floor, not an override.

FixDelete the min-font-size declaration. The rule below it already applies a 14px floor to body text; if you want an explicit floor pattern, use font-size: clamp(14px, 3.5vw, 16px) on a scoped selector.
High No horizontal-scroll safety anywhere render.py (global)

Zero overflow-x:auto declarations in the entire template. Long DPRK titles, Korean-language quotes, or wide numeric tables push the page body sideways on narrow screens. The market table and KCNA phrase table stack cleanly, but there is no fallback for anything not covered by an explicit stack rule.

FixAdd a defensive wrapper — e.g. <div style="overflow-x:auto"> around any table that isn't explicitly stacked in the mobile media query.
High Masthead H1 shrinks to 19px on mobile render.py:1304

The h1 steps from 26px desktop → 19px mobile. For a serif masthead titled "Korea Daily Brief" this is under-emphatic; the RE: line ends up nearly the same size (13px) as the title. Compare to Bloomberg/NYT mobile mastheads which stay 22–26px.

FixRaise mobile h1 to 22px; add letter-spacing: -0.2px for Georgia to sit tighter at that size.
High Section padding stays 20px 32px on 621–768px tablets render.py:1355

The tablet media query only overrides .sec, .footer { padding:14px 20px }? Actually it does — but the mobile break at 620px means the 621–768px range gets a slightly different treatment. Look at the actual output at 700px (landscape phones, small tablets) — the market strip still has 3 columns while story bodies get 20px sides. Verify visually.

FixAdd a shared .mkt-table td { padding:10px 14px } rule inside the tablet query so numbers don't collide at 621px.
Medium Dark-mode color blanket is too aggressive render.py:1368

.wrapper p, .wrapper div, .wrapper td, .wrapper span { color:#CCC !important; } flips every element to #CCC. When a callout keeps its light background (dark-mode override missed it), the text becomes near-invisible. Follow-up overrides on lines 1376–1378 patch specific cases but this is playing whack-a-mole.

FixScope the blanket to text-only containers: .wrapper .story-card p, .wrapper .story-card div etc. Or introduce a CSS custom property for text color and set it once at .wrapper.
Medium Sentiment table 48% × 2 can wrap render.py:1299

.sentiment-table td { display:inline-block; width:48%; } totals 96%. Any cell padding, border, or whitespace between cells pushes total over 100% and the second card wraps to a new line. Fine visually but the intent is a 2×2 grid, not a 1×4.

FixSet width:47% or use box-sizing:border-box plus width:50% — currently only the outer cell has box-sizing.
Medium KCNA phrase table uses table-layout:fixed without column widths render.py:1319

.kcna-dark table { table-layout:fixed } with no explicit <col> or per-cell width causes columns to divide equally regardless of content — a short phrase gets the same width as a long one. Fine until a phrase like "irreversibly and permanently terminated" needs more room.

FixSet the count column to a fixed width in HTML (width="60") so the phrase column absorbs remaining space.
Medium Chapter divider letter-spacing:5px on 9px text render.py:138

Extreme kerning at tiny size makes chapter labels harder to scan on phones. Mobile bumps font to 11px min but keeps the 5px letter-spacing — that's proportionally even more spread at the larger size.

FixDrop to letter-spacing:3px for the chapter divider; save 5px for the masthead eyebrow where it lands elegantly.
Medium Market strip creates 6 stacked rows on mobile render.py:1281 · 1350

Top mkt-table (KOSPI / Brent / KRW) and second row (BOK / CDS / GDP) both stack independently. That's six vertical rows of market data before the reader gets to Top Stories. On a 6-inch phone this is roughly two screen heights of numbers.

FixKeep 3-across on mobile with reduced font size (font-size:14px, padding:8px 4px). Numbers stay scannable and the section stays one screen tall.
Low Duplicate rules in the mobile media query render.py:1281/1352 · 1283/1347

.mkt-table td and .loc-grid td each declared twice in the same block. Second declaration wins by source order, but the redundancy makes future edits fragile — an edit to the first block silently gets overridden.

FixDelete lines 1281 and 1283; keep the later declarations. Add a comment noting the deletion so future edits don't re-introduce them.
Low 41 uses of 10px, 8 uses of 9px in output render.py (labels)

The mobile override bumps everything under 11px up to 11px, so legibility is fine. But the sheer volume of tiny-label declarations suggests the desktop design leans small — labels, eyebrows, "as of" timestamps, footnotes all sit at 9–10px, making the newsletter feel busier than it needs to.

FixDesign pass: standardize on 11px minimum for all label text, 12px for meta/timestamps. Not a mobile bug, but the reason mobile needs so many overrides.
Low RE: line at 13px next to a 19px H1 render.py:212

On mobile the H1 shrinks but the RE: line stays 13px. Hierarchy collapses — the two lines end up visually equal-weighted after the H1 fix.

FixPaired with the H1 fix above: set RE: line to font-size:12px on mobile so the type scale stays clean.
